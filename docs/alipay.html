Alipay Guide

Stripe users can sell to hundreds of millions of new customers by accepting payments through Alipay, China’s #1 payment method. This guide will introduce you to accepting Alipay payments alongside credit card payments. Let us know at alipay@stripe.com if you have any questions.

An example of Alipay checkout.

  2. Alipay in Checkout
  3. Server-side integration
  4. Additional details
  5. Reference

Overview

China is the world’s largest e-commerce market, and recent years have seen an enormous increase in traffic between Chinese customers and online sellers outside of China. The most popular Chinese payment method is the Alipay consumer wallet, and Stripe has partnered with Alipay so that any Stripe user can accept Alipay payments.

Alipay payments are unified with all other payment types in the API and Dashboard. An existing Stripe user can start accepting Alipay payments without any registration process and without modifying any backend code.

Historically, Alipay customers would have been redirected to alipay.com in order to make a payment. However, Stripe enables on-site payment acceptance through Checkout: there is no redirect flow, and Stripe users see a correspondingly high conversion rate.

Note on transfers

Currently, Alipay payments can only be paid out in USD to USD-denominated bank accounts.

Alipay users pay in RMB, and Stripe automatically handles currency conversion to USD. You do not need a Chinese entity or bank account to use Alipay. Stripe users from any country can accept Alipay payments as long as they have a USD-denominated bank account. The default fee for Alipay payments matches that for credit card payments: 2.9% + 30 cents. Please contact us if you’d like to discuss custom pricing or if you’d like to hear from us when we support bank accounts outside the United States.

The following sections describe how to enable Alipay payments on your site. If you have any questions, please let us know by emailing alipay@stripe.com or joining us in the #stripe channel on Freenode.

Alipay in Checkout

To accept Alipay payments, load Checkout using the following code. Notice the data-alipay="true" option. This tells Checkout to accept both credit card charges (the default) and Alipay charges. If you are already a Checkout user, this one line is the only code change you need to make to accept Alipay!

<form action="/create_payment" method="POST">
  <script
    src="https://checkout.stripe.com/checkout.js" class="stripe-button"
    data-key={PUBLISHABLE_API_KEY}
    data-image="/square-image.png"
    data-name="Demo Site"
    data-description="2 widgets ($20.00)"
    data-amount="2000"
    data-currency="usd"
    data-alipay="true">
  </script>
</form>

Instead of passing data-alipay="true", you can pass data-alipay="auto" in which case the Alipay option will be shown to customers likely to have an Alipay account.

You can pass data-locale="auto" to display Checkout in the user’s preferred language, if available. Most Alipay users will find Checkout easier to use when in Simplified Chinese.

Checkout displays as follows when data-alipay="true":

We use SMS verification to authenticate the Alipay user. You can try this flow with your test mode API key (or your live mode key if you have a supported Alipay account). In test mode, the correct value of the SMS code is fixed as 123456.

For some payments, Stripe will dynamically request the last five digits of the customer’s national ID number (“last five”) as an additional factor of authentication. In test mode, we always ask for the last five, and it is fixed as: 12345.

Reusable and non-reusable account access

By default, you must pass amount and currency parameters when using Alipay with Checkout. You are asking for the customer’s permission to create a one-time payment for this amount and currency. The verification SMS will inform the customer of the amount and currency. This is a great feature from the customer’s perspective: they can pay online without giving away permanent access to their account.

If you need reusable access to the customer’s account (to create recurring payments, for instance), then you must pass data-alipay-reusable="true" to Checkout. Checkout will inform the customer that you have requested reusable access (you can see this by trying out the flow in test mode). Alipay users are accustomed to only allowing reusable access in certain circumstances. You should only ask for this access if there is a real need for it, and if the need is clear to the user.

Your choice here is Alipay-specific and does not affect your credit card configuration.

When viewing an Alipay account through the API, you can see whether you have reusable access. Here is an account with reusable access:

"alipay_account": {
  "id": "aliacc_4VlyMzbhx5cMJ4",
  "object": "alipay_account",
  "livemode": false,
  "created": 1406916427,
  "fingerprint": "VcfP7nsUODDwkd5M",
  "used": false,
  "reusable": true,
  "payment_amount": null,
  "payment_currency": null
}

If you do not have reusable access then reusable=false, and payment_amount/payment_currency indicate the amount and currency of the payment that you are allowed to create.

All Alipay account objects have a used attribute that indicate whether your access has been used.

Server-side integration

Charges and customers

Prior to using Alipay, you probably used Tokens of type card. Now you will also use Tokens of type alipay_account. These new Tokens, which are returned by Checkout, have an alipay_account attribute in place of a card attribute. Like all Tokens, they can either be passed to /v1/charges/ to create a Charge or to /v1/customers to save on a Customer.

Creating a Charge

(See the API reference.)

Here is an example of creating a Charge from an Alipay account:

curl https://api.stripe.com/v1/charges \
   -u sk_test_BQokikJOvBiI2HlWgH4olfQ2: \
   -d amount=2000 \
   -d currency=usd \
   -d source=atok_4XNshPRgmDRCVi

Notice that the Token’s ID is passed as the source parameter. This parameter behaves identically to the card parameter, and you can continue using the card parameter for compatibility with an older Stripe integration. The Alipay account will be returned as the source attribute of the Charge. For more information on the source parameter and source attribute, see our blogpost on the topic.

Attaching to a Customer

(See the API reference.)

Here is an example of attaching an Alipay account to a Customer:

curl https://api.stripe.com/v1/customers \
   -u sk_test_BQokikJOvBiI2HlWgH4olfQ2: \
   -d source=atok_4XNshPRgmDRCVi

The Alipay account will be the customer’s default_source and will appear in their sources list.

Customers with attached Alipay accounts can be used for subscription billing. To do this, make sure that you have reusable Alipay accounts.

Test mode tokens

In test mode, you can create Tokens directly through the API rather than using Checkout manually. You can do this using your secret or publishable test mode key. For instance:

curl https://api.stripe.com/v1/tokens \
   -u sk_test_BQokikJOvBiI2HlWgH4olfQ2: \
   -d email="maidongxi@example.com" \
   -d alipay_account[reusable]=true

The alipay_account parameter supports the following keys:

  • reusable: Required. Either true or false.
  • alipay_username: Optional. Default to the email address on the Token.
  • payment_amount and payment_currency: Provided if and only if reusable=false.

Additional details

Bookkeeping

The Dashboard provides centralized bookkeeping for all of your payment types (including credit card and Alipay). You can view Alipay charges alongside credit card charges in your payments list and balance history. Both of those pages support CSV export.

Refunds

You can refund any charge while viewing it in the dashboard. You can also create a refund through the API:

curl https://api.stripe.com/v1/charges/py_4vEAo8YvLDL6sE/refunds \
   -u sk_test_BQokikJOvBiI2HlWgH4olfQ2: \
   -X POST

Supported Alipay accounts

Alipay users pay in RMB, which we convert to USD to pay out to you. Because of FX restrictions on RMB, Alipay users can only use Checkout if their Alipay account is verified (i.e. linked to a real-world identity). The Alipay user must have proved their identity as a citizen of mainland China by uploading an image of their national ID card.

If you are curious about the Alipay user-experience, you can create an account without being a citizen of mainland China, but you can only pay through Checkout if you are.

The only other requirements on the customer’s Alipay account are that it has a linked mobile number and that it is capable of funding the payment.

Declines and disputes

As with credit cards, Alipay or Stripe can decline payments, in which case you will receive a 402 error from /v1/charges.

We’re happy to say that you will not receive a dispute if a customer’s Alipay account is used illicitly. Alipay and Stripe will handle such issues internally. In the Alipay context, payments will only be disputed if the customer has a complaint with the provided goods or service.

To prevent abuse of stolen Alipay accounts, Stripe sets amount-limits on the payments that can be created by each Alipay user. We will set the limits high enough that they do not interfere with your customers but prevent abnormal use. The default amount-limits are as follows:

  • 750.00 USD per payment
  • 2,250.00 USD per customer per month

Please reach out if your Stripe account needs higher limits.

Reference

This reference covers alipay_account API objects. See the main reference for information on all other endpoints and objects.

Alipay account API objects are possible sub-objects for Charges, Tokens, and Customers. An example Alipay account is:

"alipay_account": {
  "id": "aliacc_4XNszNnmyohLqD",
  "object": "alipay_account",
  "livemode": false,
  "created": 1407288079,
  "fingerprint": "VcfP7nsUODDwkd5M",
  "used": true,
  "reusable": false,
  "payment_amount": 1000,
  "payment_currency": "usd"
}

The attributes are:

  • fingerprint: Similarly to the fingerprint on card objects, this fingerprint uniquely identifies the account and will be the same across all Alipay account objects that are linked to the same Alipay account.
  • used: Whether this Alipay account object has ever been used for a payment.
  • reusable: True if you can create multiple payments using this account. If the account is reusable, then you can freely choose the amount of each payment.
  • payment_amount and payment_currency: If reusable=false, then these attributes are non-null and constrain the payment that you can create from the account.
